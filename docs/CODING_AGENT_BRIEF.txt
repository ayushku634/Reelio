Reelio Chrome Extension


What You Are Building
A Chrome Extension called Reelio. It intercepts Instagram's internal API calls to capture saved reels from https://www.instagram.com/ayush.hs26/saved/all-posts/, enriches each reel with a spoken transcript and metadata, stores everything locally in IndexedDB, and lets the user search their saved reels using natural language via a popup UI.

The Single Most Important Thing
Everything runs locally in the browser. No backend. No API keys. No external database. Zero cost to run. All AI models run via WebAssembly in Web Workers.
The Most Critical Implementation Detail - Read Before Anything Else
3-Layer Interception Architecture - Non-Negotiable
Chrome MV3 content scripts run in an ISOLATED JavaScript world. They CANNOT patch window.fetch as Instagram sees it. A common mistake is trying to patch fetch directly in content.js - this silently fails.  The CORRECT 3-layer approach:  LAYER 1 - public/page-interceptor.js   Ģ A static JS file declared in web_accessible_resources   Ģ Patches window.fetch and XMLHttpRequest in Instagram's JS context   Ģ Dispatches CustomEvent("reelioApi", { detail: { url, body, status } })   Ģ Has idempotency guard: if (window.__reelioInterceptor) return;  LAYER 2 - content.js   Ģ Runs at document_start   Ģ Injects page-interceptor.js via: script.src = chrome.runtime.getURL("public/page-interceptor.js")   Ģ Listens: window.addEventListener("reelioApi", evt => chrome.runtime.sendMessage(...))   Ģ NO business logic - pure relay  LAYER 3 - background.js (Service Worker)   Ģ Receives { type: "API_PAYLOAD", payload: { url, body } }   Ģ Parses body as JSON, fingerprints, deduplicates, runs pipeline  Do NOT skip or merge any layer.

Popup UI Spec
The popup should be 380px wide. Clean, minimal, dark-friendly. Three views:

View 1: Search (default)
Ģ Search input - full width, autofocused on popup open
Ģ Top N selector - segmented control: [3] [5] [10] [20]
Ģ Result cards - thumbnail (64ū64), creator handle, caption snippet (1 line, truncated), category badge, score %, Open button
Ģ Below main results: "N partial matches from older reels" expandable section
Ģ Empty state: "Go to instagram.com/ayush.hs26/saved/all-posts/ and scroll to start indexing"

View 2: Loading (first launch only)
Ģ "Setting up search engine" progress bar (25MB model download)
Ģ Shown once, never again after model is cached

View 3: Settings (gear icon)
Ģ Full context reels: slider 50-500, current value shown
Ģ Storage used: "3.4 MB used"
Ģ Visual search: toggle (off by default, shows 1.8GB warning on enable)
Ģ Vision queue: "8 reels queued for visual analysis" (only shown when queue > 0)
Ģ [Clear All Data] - red button, confirmation required
