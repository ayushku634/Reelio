2. Manifest V3 Configuration
manifest.json (complete)
{   "manifest_version": 3,   "name": "Reelio",   "version": "1.0.0",   "permissions": [     "scripting",     "storage"   ],   "host_permissions": [     "https://www.instagram.com/*",     "https://*.cdninstagram.com/*"   ],   "background": {     "service_worker": "background.js",     "type": "module"   },   "content_scripts": [{     "matches": ["https://www.instagram.com/*"],     "js": ["content.js"],     "run_at": "document_start"   }],   "action": {     "default_popup": "popup.html",     "default_icon": "assets/icons/48.png"   },   "web_accessible_resources": [{     "resources": [       "public/page-interceptor.js",  // PAGE CONTEXT injection       "embedding-worker.js",       "vision-worker.js"     ],     "matches": ["https://www.instagram.com/*"]   }] }
Why "scripting" permission?
"scripting" is required alongside "storage" in MV3. The page-interceptor.js file must be declared under web_accessible_resources so content.js can load it via chrome.runtime.getURL() and inject it as a <script> tag into the Instagram page DOM. This is the only compliant way in MV3 to run code in the page's JavaScript context (not the isolated content script world). The transformers.js and Moondream2 WASM binaries must also be bundled via webpack - they cannot be loaded from external CDNs at runtime due to MV3 CSP restrictions.
3. Module Specifications
3.1 public/page-interceptor.js
This is the most critical file. It runs in Instagram's own JavaScript context (not the isolated content script world) and patches window.fetch and XMLHttpRequest. It is NOT bundled by webpack - it is served as a static file from public/ and injected via a <script> tag by content.js.
• IIFE with idempotency guard (window.__reelioInterceptor) - safe to re-inject
• Patches window.fetch - clones response before reading, returns original to Instagram
• Patches XMLHttpRequest.prototype.open and .send - captures response on load event
• Dispatches CustomEvent("reelioApi") with { url, body, status } - bridges to content script
• Filters only URLs containing "/graphql" or "/api/" before dispatching
• All errors silently swallowed - never throws, never breaks Instagram
• Total size target: under 3KB unminified

3.2 content.js
Injected at document_start. Two responsibilities only: (1) inject page-interceptor.js into the page, (2) relay CustomEvents from page context to background Service Worker.
• Injects public/page-interceptor.js via document.createElement("script") into document.head
• Script URL obtained via chrome.runtime.getURL("public/page-interceptor.js")
• Listens for window CustomEvent("reelioApi")
• On event: chrome.runtime.sendMessage({ type: "API_PAYLOAD", payload: evt.detail }) - fire and forget
• No business logic, no parsing - pure relay layer
• Total size target: under 1KB

3.2 interceptor.js
Pure function module. Takes raw GraphQL JSON, returns array of raw reel objects or empty array.
• Uses structural duck-typing to identify saved reel responses
• Normalises nested GraphQL edge/node structure into flat reel array
• Extracts: id, shortcode, video_url, thumbnail_url, accessibility_caption, video_subtitles_uri, owner, caption, audio_track, hashtags, taken_at
• Returns [] for any unrecognised response shape - never throws

3.3 cc-fetcher.js
Async module. Takes a .vtt URL, returns clean transcript string or null.
.vtt Parsing Logic
1. fetch(vttUrl) ? raw text 2. Split by line 3. Filter out: "WEBVTT" header, timestamp lines (regex: /\d{2}:\d{2}/), blank lines 4. Join remaining lines with space 5. Deduplicate consecutive duplicate phrases (common in auto-captions) 6. Trim and return
3.4 metadata-tagger.js
Pure synchronous function. Takes raw reel object, returns enriched metadata fields.
Category detection uses keyword dictionaries per category:

3.7 search.js
Core search logic. Called by background.js message handler.
1. Receive query string
2. Post to Embedding Worker: { type: "EMBED", text: query }
3. Await EMBED_RESULT ? queryVector (Float32Array)
4. Load all EmbeddingRecords from db.getAllEmbeddings()
5. For each record: score = cosineSimilarity(queryVector, record.vector)
6. Sort by score descending
7. Take top N (from user settings)
8. Load corresponding ReelDocuments from IndexedDB
9. Return array of {document, score} sorted by score
