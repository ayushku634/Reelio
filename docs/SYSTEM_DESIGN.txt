Reelio Chrome Extension


1. System Overview
Reelio is a fully client-side system. There is no backend, no server, no cloud database. Every component - data collection, enrichment, storage, embedding, and search - runs inside the user's Chrome browser. The only outbound network calls are to Instagram's own CDN (to fetch .vtt subtitle files and thumbnail images, which the browser would fetch anyway during normal use).

Architecture Principle
Local-first. The system is designed so that once the one-time model downloads are complete, the extension works with zero network dependency. Search is fully offline. This is both a privacy guarantee and a performance guarantee.

4. Network Interception Design
4.1 Three-Layer Architecture (Page ? Content ? Background)
Reelio uses a 3-layer interception architecture. This is the correct approach for Chrome MV3 because content scripts run in an isolated world - they cannot directly access or patch window.fetch as seen by Instagram's own JavaScript. The solution is to inject a separate page-context script that runs in Instagram's JavaScript environment.

Architecture Flow
+- Page Context (public/page-interceptor.js) -+ Î  Patches window.fetch + XMLHttpRequest       Î Î  Dispatches CustomEvent("reelioApi") w/ data Î +---------------------------------------------+                       Î                       ? +- Content Script (content.js) ---------------+ Î  Injects page-interceptor.js via <script>   Î Î  Listens for CustomEvents, forwards to BG   Î +---------------------------------------------+                       Î                       ? +- Background Service Worker (background.js) -+ Î  Parses JSON, deduplicates, runs pipeline,  Î Î  writes to IndexedDB, updates badge         Î +---------------------------------------------+
4.2 Why a Page-Injected Script (Not Direct Content Script Patching)
Content scripts in Chrome MV3 operate in an "isolated world" - they share the DOM with the page but have a completely separate JavaScript environment. This means window.fetch in a content script is NOT the same fetch Instagram calls. Patching it there would intercept nothing from Instagram's own network calls.
The solution: inject a .js file from web_accessible_resources directly into the page's <head> as a <script> tag. This file runs in the page's JavaScript context and can successfully patch window.fetch and XMLHttpRequest as Instagram sees them.

page-interceptor.js - Runs in Page Context
(function () {   if (window.__reelioInterceptor) return; // idempotent guard   window.__reelioInterceptor = true;    // -- Patch fetch --   const originalFetch = window.fetch;   window.fetch = async function (input, init) {     const resp = await originalFetch.apply(this, arguments);     try {       const url = typeof input === "string" ? input : input.url || "";       if (url.includes("/graphql") || url.includes("/api/")) {         const cloned = resp.clone(); // CRITICAL: clone before reading         const text = await cloned.text();         window.dispatchEvent(new CustomEvent("reelioApi", {           detail: { url, body: text, status: resp.status }         }));       }     } catch (e) { /* silent - never break Instagram */ }     return resp; // return ORIGINAL to Instagram   };    // -- Patch XHR (Instagram uses both) --   const origOpen = XMLHttpRequest.prototype.open;   const origSend = XMLHttpRequest.prototype.send;   XMLHttpRequest.prototype.open = function (method, url) {     this._reelioUrl = url;     return origOpen.apply(this, arguments);   };   XMLHttpRequest.prototype.send = function () {     this.addEventListener("load", function () {       if (this._reelioUrl?.includes("/graphql")) {         window.dispatchEvent(new CustomEvent("reelioApi", {           detail: { url: this._reelioUrl, body: this.responseText, status: this.status }         }));       }     });     return origSend.apply(this, arguments);   }; })();
content.js - Bridges Page Context to Extension
// Step 1: Inject the page-context interceptor script const scriptUrl = chrome.runtime.getURL("public/page-interceptor.js"); const script = document.createElement("script"); script.src = scriptUrl; script.type = "text/javascript"; document.head.appendChild(script);  // Step 2: Listen for intercepted API data and relay to Service Worker window.addEventListener("reelioApi", (evt) => {   chrome.runtime.sendMessage({ type: "API_PAYLOAD", payload: evt.detail }); });
4.3 Manifest Changes Required for This Pattern
The page-interceptor.js file must be declared as a web_accessible_resource so the content script can inject it via chrome.runtime.getURL(). The "scripting" permission is also needed.
manifest.json additions
"permissions": ["scripting", "storage"], "host_permissions": [   "https://www.instagram.com/*",   "https://*.cdninstagram.com/*" ], "web_accessible_resources": [{   "resources": ["public/page-interceptor.js"],   "matches": ["https://www.instagram.com/*"] }]
4.4 Response Fingerprinting (isSavedReelsResponse)
The background script must identify which intercepted responses contain saved reel data. Use structural duck-typing - NOT Instagram's internal doc_id (which changes frequently). The target URL for Reelio is https://www.instagram.com/ayush.hs26/saved/all-posts/
 Check URL contains "/graphql" AND current tab URL matches instagram.com/ayush.hs26/saved/
 Check parsed JSON contains an edges array whose nodes have shortcode or video_url fields
 Check nodes contain accessibility_caption or video_subtitles_uri (signals reel content)
 Return [] for any unrecognised shape - never throw

Background: deduplicate and process
const processedIds = new Set(); // in-memory dedup within session  chrome.runtime.onMessage.addListener((msg, sender) => {   if (msg.type === "API_PAYLOAD") {     try {       const json = JSON.parse(msg.payload.body);       const reels = extractReels(json); // returns RawReel[]       const newReels = reels.filter(r => !processedIds.has(r.id));       newReels.forEach(r => processedIds.add(r.id));       if (newReels.length) runEnrichmentPipeline(newReels);     } catch (e) { /* non-JSON or irrelevant - silent */ }   } });
4.5 CC File Fetching
Once a reel's .vtt subtitle URL is extracted from the GraphQL response, the Service Worker fetches it directly:
 Fetch is made from Service Worker context (not content script) to avoid CORS issues
 cdninstagram.com host_permission in manifest grants this access
 Response is plain text - parse by stripping timestamp lines and joining cue text
 If fetch fails (expired CDN URL, missing captions), transcript field is set to null - graceful degradation

5. Embedding & Search Design
5.1 Model Loading
 transformers.js loaded as ES module in Embedding Worker
 Model all-MiniLM-L6-v2 downloaded from Hugging Face CDN on first use
 Cached to browser cache storage after first download - subsequent loads are instant
 Worker remains alive across popup open/close cycles via Service Worker keep-alive ping

5.2 Cosine Similarity Implementation
Pure JS cosine similarity - no library needed
function cosineSimilarity(a, b) {   let dot = 0, magA = 0, magB = 0;   for (let i = 0; i < a.length; i++) {     dot += a[i] * b[i];     magA += a[i] * a[i];     magB += b[i] * b[i];   }   return dot / (Math.sqrt(magA) * Math.sqrt(magB)); }
5.3 Search Complexity
For 500 reels with 384-dimension vectors: 500 æ 384 = 192,000 multiply-add operations. This completes in under 5ms on any modern CPU. Linear scan is perfectly adequate at this scale - no approximate nearest neighbour index (HNSW etc.) is needed.

6. Vision Pipeline Design
6.1 Frame Extraction
 Three frame URLs are derived from the reel's CDN thumbnail URL pattern
 Cover frame (00:00) - always available as the poster image
 Mid-reel frame - fetched from CDN if URL pattern supports it
 Late frame - fetched from CDN if URL pattern supports it
 Fallback: if only cover frame available, single-frame analysis still runs

6.2 Moondream2 Prompt
Vision Prompt Template
Describe what is shown in this Instagram Reel frame. Be specific about: - The setting (indoor/outdoor, location type) - What the person is doing or showing - Any visible objects, food, equipment, text on screen - The overall vibe and aesthetic Keep your description to 2-3 sentences. Write as if helping someone remember why they saved this reel.
6.3 Queue Management
 Vision processing runs in a FIFO queue in the Vision Worker
 New reels are appended to the queue as they are indexed
 Queue persists in memory - if popup closes, processing continues in Service Worker
 Processed reels have their visualDescription field updated in IndexedDB
 Queue depth shown in settings panel: "Visual analysis: 12 reels remaining"

Reel still indexed with other layers.Vision model not downloadedvisualDescription = null. All other search still works.Embedding model unavailableSearch falls back to keyword matching on stored document text.IndexedDB quota exceededOldest Tier 1 documents demoted to Tier 2 before writing new ones.Duplicate reel encounteredExisting record updated with any newly available enrichment (e.g. new CC data).
